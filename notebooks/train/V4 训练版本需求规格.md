  一、核心修复：解决后验坍缩（P0，阻塞项）

  1.1 KL 退火调度（KL Annealing）

  ▎ 目标：训练初期不施加 KL 压力，先让编码器学会编码有用信息，后期再逐步引导潜空间规范化。

  具体需求：
  - KL 权重从 0 开始，在前 N 个 epoch 内线性增加到目标值（如 β=1.0 或 β=0.1）
  - 支持三种调度策略：线性、sigmoid、cyclic
  - 超参可配置：起始权重、目标权重、升温 epoch 数、调度曲线类型

  1.2 Free Bits 机制

  ▎ 目标：设 KL 下限，低于阈值不惩罚，防止编码器把 KL 压到接近 0 就"收手"。

  具体需求：
  - 对每个潜维度独立设 KL 下限，例如 λ=0.1
  - 当某维度的 KL < λ 时，该项不参与损失计算
  - 这样编码器至少需要用 λ 的信息量来编码输入

  1.3 坍缩检测指标

  ▎ 目标：训练过程中实时监控，坍缩发生时立即看到。

  具体需求：
  - 每个 epoch 结束记录：
    - mu_norm：μ 的 L2 范数（均值、方差），接近 0 说明编码器躺平
    - sigma_mean：σ 的均值，接近 0 说明方差坍缩
    - kl_per_dim：每个潜维度的平均 KL 值，找"死亡维度"占比
    - z_mutual_info：z 和输入之间的互信息估计
  - 这些指标写入 eval_reports/experiment_meta_*.json

  1.4 语义一致性验证

  ▎ 目标：确保换房间数量后，编码器的 μ 确实产生了可感知的变化。

  具体需求：
  - 每 N 个 epoch 做一次语义插值测试：
    - 构造两个样本，仅卧室数不同（3 vs 5），其他条件相同
    - 计算两个 μ 的余弦距离，记录变化幅度
    - 如果余弦距离 < 阈值（如 0.01），标记为"坍缩警告"
  - 同理测试 site 尺寸变化对 μ 的影响

  ---
  二、超节点优化：让模型从数据中学习房间分组

  2.1 房间类型可学习 Embedding

  ▎ 目标：不做硬 3 分类，让 11 种房间类型各自有一个可学习的 embedding 向量，模型在训练中发现哪些类型在连接模式上相似。

  具体需求：
  - 11 种房间类型 → 11 个可学习 embedding（维度如 32）
  - 每个房间节点的初始特征 = 几何特征（8 维）+ 类型 embedding（32 维）拼接
  - 训练后分析 embedding 的余弦相似度矩阵，验证是否自然形成了 living/service/circulation 三个簇
  - 如果三个簇自然形成 → 验证了建筑常识
  - 如果形成了不同的簇（如 4-5 类）→ 可能是更有意义的发现

  2.2 节点同构化（可选方案）

  ▎ 目标：用同构 GNN + 注意力替代异构 GNN，彻底摆脱硬编码超节点。

  具体需求：
  - 所有房间节点统一类型，用 GAT（图注意力网络）或 TransformerConv
  - 边类型保留 horizontal / vertical 两种关系
  - 节点的类型 embedding + 几何特征作为输入，注意力自动学习哪些节点类型之间需要传递更多信息
  - 对比：同构注意力 GNN vs 异构 SAGEConv + 可学习 embedding，看哪个更好

  三、默认尺寸优化：让输入条件自适应

  3.1 训练数据统计替代写死常量

  ▎ 目标：不再用写死的 DEFAULT_ROOM_SIZE，而是从训练数据集中统计每个房间类型的真实尺寸分布。

  具体需求：
  - 预处理时计算每个房间类型的：
    - 面积中位数和四分位距（IQR）
    - 长宽比中位数
    - 体积中位数
  - 生成时每个房间的初始尺寸从该类型的统计分布中随机采样（而非固定值）
  - 采样时加入场地约束：如果 site_x × site_y < 某阈值，按比例缩小

  3.2 场地自适应缩放

  ▎ 目标：同一套房间清单，在 30m×20m 和 10m×8m 的场地上，初始尺寸应该不同。

  具体需求：
  - 定义基准场地（如 18m×15m = 270㎡）
  - 实际场地面积 / 基准场地面积 = 缩放因子（上下限限定在 0.6~1.4）
  - 初始房间尺寸 × 缩放因子 → 自适应初始尺寸
  - 缩放因子同时影响 spring layout 的 k 参数（节点排斥力），避免小场地节点挤成一团

  3.3 房间特征增强

  ▎ 目标：让每个房间节点的特征包含"相对场地"的信息，而不仅是绝对数值。

  具体需求：
  - 当前特征：面积、长宽比、中心位置（归一化到场地）
  - 新增特征：面积占场地百分比、房间体积占比、是否转角房间（靠近场地边界 1 个体素内）、是否为端部房间（只有一个邻居）

  ---
  四、条件向量强化：让解码器"听见"需求变化

  4.1 条件向量维度扩容

  ▎ 目标：让条件信号的"音量"足够大，不被 256 维潜空间淹没。

  具体需求：
  - 当前条件向量：17 维（3 场地 + 11 房间占比 + 3 采光）
  - V4 增加到 64-128 维：
    - 3 维场地归一化尺寸
    - 11 维各房间类型实际个数（非占比，保留绝对值）
    - 3 维采光统计
    - N 维 → 通过一个小型 MLP 将 17 维投影到 64 维
  - 或采用 FiLM（Feature-wise Linear Modulation）：条件向量过 MLP 产生 scale + shift 参数，注入解码器每一层

  4.2 Cross-Attention 注入

  ▎ 目标：不在全连接层一次性把条件混进去，而是在解码器每一层都"提醒"模型条件是什么。

  具体需求：
  - 解码器 4 层 ConvTranspose3d 之间插入 cross-attention 层
  - Query = 该层的特征图展平
  - Key/Value = 条件向量的投影
  - 让每个空间位置的体素都能"查看"整体条件

  ---
  五、解码器架构升级

  5.1 自注意力层

  ▎ 目标：打破纯卷积的局部感受野限制。

  具体需求：
  - 在第 2 层或第 3 层 ConvTranspose3d 后面插入一个轻量 self-attention（如 axial attention 或 window attention）
  - 显存受限的话用 linformer 或 performer 等线性注意力
  - 目标：体素网格左上角的卧室能"意识到"右下角的客厅

  5.2 输出层结构约束

  ▎ 目标：让输出的房间数尽可能接近输入要求的数量。

  具体需求：
  - 解码器最后额外的输出头：11 维向量，预测每种房间类型的是否存在（二分类交叉熵辅助损失）
  - 辅助损失 × 0.1 权重加入总损失
  - 不强制连通域数量匹配（太难），但至少让解码器"知道"要生成几种房间

  ---
  六、推理管线升级

  6.1 批量化采样

  ▎ 目标：K 次采样并行跑，不用串行循环。

  具体需求：
  - infer_voxels 里 K 个 z 拼成 batch（K, 256），一次 decoder forward 得到所有结果
  - K=8 时预期提速 3-5 倍

  6.2 潜空间梯度优化（可选）

  ▎ 目标：用梯度上升找更好的 z，而非纯随机撒点再挑。

  具体需求：
  - 以评分函数为目标，对 z 做梯度上升（固定解码器权重）
  - 迭代 50-100 步，学习率 0.01
  - 随机采样 + 梯度精修结合：先撒 4 个随机点，每个做 25 步优化，选最优

  6.3 评分函数可配置化

  ▎ 目标：评分权重和阈值从配置文件读，不写死在代码里。

  具体需求：
  - scoring_config.json：包含各扣分项的权重、面积限制表、碎片惩罚系数等
  - 支持按项目/数据集切换不同的评分配置

  ---
  七、训练数据多样性保障

  7.1 数据覆盖检查

  ▎ 目标：确保训练集不是"80% 都是 3 卧室"。

  具体需求：
  - 统计训练集中各房间类型数量的分布（直方图）
  - 如果某配置占比超过 30% → 做下采样
  - 人工构造极端配置样本做数据增强（如：只有 1 个卧室、6 个卧室）

  7.2 数据增强

  ▎ 目标：让模型见过更多"异类"布局。

  具体需求：
  - 场地尺寸随机缩放（± 30%）
  - 房间数量随机增减（在合理范围内）
  - 镜像翻转（增强方位不变性）
  - 保留原始标注，不做有损增强

  ---
  八、训练实验记录增强

  8.1 坍缩相关指标

  ▎ 目标：每次实验自动记录坍缩诊断信息。

  具体需求：
  - 在 experiment_meta_*.json 中新增字段：
    - kl_annealing_schedule：KL 权重调度参数
    - final_mu_norm_mean / std
    - final_sigma_mean
    - final_kl_per_dim：256 维的 KL 值列表
    - dead_dims_ratio：KL < 0.01 的维度占比
    - semantic_sensitivity：换卧室数后的 μ 余弦距离
