# V4 实现方案 · 诊断结论与设计

> 配套需求规格：`V4 训练版本需求规格.md`。本文档记录**对规格的评估结论**、**诊断实证**、以及**最终落地的 V4-b 设计**。生成器：`scripts/notebook_build/create_v4_train.py`（源=true）。

## 一、对原始需求规格的评估结论

原规格把"后验坍缩（KL 退火 + Free Bits）"列为 P0 阻塞项。**诊断证明这是误判**：当前 V3 没有后验坍缩，真正病灶是**条件被淹没**。因此 V4 主攻条件机制，而非 KL/FreeBits。

被推迟/否决的规格项：
- KL 退火 / Free Bits（1.1/1.2）：诊断显示 KL 健康，硬上可能更糟 → **不做**。
- 同构 GNN（2.2）、解码器自注意力 / Cross-Attention（4.2/5.1）：小数据(~468)上加容量易过拟合 → **推迟**，FiLM 够用再说。
- 潜空间梯度优化（6.2）：评分函数不可微 → **搁置**。
- 数据下采样（7.1）：样本本就稀缺 → 改为**过采样/加权**（后续）。

## 二、诊断实证（V3 最佳权重，Step D / diagnose_v4a.py）

| 组 | 关键数值 | 结论 |
|---|---|---|
| A 坍缩 | `KL_total=33.8`、`mu_norm=7.6`、`σ≈0.94`、活跃单元 37.9% | **非后验坍缩**，z 携带信息 |
| B 编码器语义 | 卧室 3↔5 μ 余弦距离 `0.0001`、场地 `0.0000` | 编码器 μ 对条件**几乎无响应**（mean-pool 稀释） |
| C 解码器条件 | n_occ 健康(~22–23k)；改 cond `0.034%`、抹零 cond `1.27%`、改 z `1.37%`；cond/z=`0.025` | **非塌空**；**条件被彻底淹没**；解码器整体模板化 |

## 三、根因（对照 `spatial_modal_infer/model.py`）

1. **入口淹没**：`cond(17) concat z(256)`，cond 仅占输入 6% 且数值幅度小（占比 0~1），z 范数 7.6。
2. **只注入一次**：4 层 `ConvTranspose3d` 之后无再注入。
3. **BatchNorm3d 抹平条件**：batch=4 下 BN 逐通道归一化，削掉 fc 注入的条件偏移。
4. **编码器 mean-pool 稀释**：`global_mean_pool` 把节点平均，房间数变化被抹平。
5. **占比而非绝对数**：`stats/total` 让 3卧 vs 5卧 数值差变小。

## 四、V4-b 设计（最终落地）

### 4.1 条件向量（`build_condition_vector`，graph_utils 同步）
- 房间维度：占比 → **绝对数量**，软缩放 `count / COND_COUNT_SCALE`（=4.0；4 间→1.0，保留 3vs5 差异）。
- 采光 3 维同样改绝对数量软缩放。
- 维度仍为 17（3 场地 + 11 房间 + 3 采光），语义改为绝对。

### 4.2 条件升维 + FiLM 注入（解码器，核心）
- `CondEncoder`：`Linear(17→64)+LayerNorm+SiLU` ×2 → `cond_embed(64)`。
- 解码器 4 层去掉 `BatchNorm3d`，改 `GroupNorm(affine=False)` + **FiLM**：每层 `x = norm(x)*(1+γ)+β`，`(γ,β)=Linear(cond_embed)`。γ 初始≈0 → 初始等于恒等，训练稳定。
- `fc` 输入改为 `z(256) + cond_embed(64)`。
- **解码器签名不变** `decoder(z, cond_raw)`（内部做 embed）→ 推理包 `pipeline.py` 调用无需改。

### 4.3 编码器计数感知 pooling（修 B）
- `_pool` 在 mean-pool 基础上，拼接**每超类节点计数**（living/service/circulation，软缩放）→ `to_mu/to_logvar` 输入维 `hidden+3`。

### 4.4 计数辅助损失（弱监督，修 B + 让 z 编码程序）
- `count_head: Linear(256→128→11)`，从 `mu` 预测各房间类型数量。
- `aux = smooth_l1(count_head(mu), target_counts)`，`target_counts = cond[:,3:14]*COND_COUNT_SCALE`。
- 总损失：`bce + DICE_WEIGHT*dice + kl_weight*kl + AUX_COUNT_WEIGHT*aux`（`AUX_COUNT_WEIGHT=0.1`）。
- `model.forward` 仍返回 `(logits, mu, logvar)`，aux 经 `model.predict_counts(mu)` 单独算 → Step D / 探针 / forward_model 不受影响。

### 4.5 损失其余
- BCE 不加 pos_weight（n_occ 健康，无塌空）。KL 维持 `KL_WEIGHT_MAX=5e-3 / anneal=40`（坍缩非问题，保持不动以隔离变量）。
